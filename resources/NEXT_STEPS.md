# 🎓 After the Capstone — Where to Go Next

> You finished a hard 24-week course. Rest for a weekend. Then pick one path — or invent your own.

---

## Path A: Go deep on AI Test Engineering (your track)

You just built the thing. Now become the person who builds the next version.

**Month 1 after course:**
- Polish the capstone. Fix issues users report.
- Write a second blog post: "6 months in — what I got wrong about AI-powered QA"
- Contribute a feature to Playwright, Promptfoo, or an AI test tool

**Months 2–3:**
- Propose a talk at a QA or AI engineering meetup
- Start a tiny side project: AI-enhanced visual regression testing
- Read [AgentBench](https://github.com/THUDM/AgentBench) — think about what "QA" means for agents

**Months 4–6:**
- Apply for roles specifically targeting AI testing
- Consider: Testim, Mabl, Applitools, Microsoft (Copilot testing teams), LLM eval startups

## Path B: Become an AI Security / Red Team specialist

If Module 8 was your favorite, this niche is booming and under-staffed.

**Certifications worth pursuing:**
- [OWASP Certified AI Tester](https://owasp.org/) (when available)
- [SANS AI Security courses](https://www.sans.org/)
- [CISM / CISSP](https://www.isc2.org/) — general security foundation, broader value

**Communities:**
- Embrace the Red blog + Discord
- HackerOne's AI vulnerability programs
- Anthropic's [bug bounty](https://www.anthropic.com/responsible-disclosure-policy)

**Target roles:**
- AI Red Team @ Anthropic / OpenAI / Microsoft
- AI Security Engineer @ Lakera, Protect AI, HiddenLayer
- General security engineer with AI specialization

## Path C: Deep dive into Agent Evaluation & Benchmarking

The most academically interesting part of AI engineering in 2026.

**Papers/benchmarks to work through:**
- [SWE-bench](https://www.swebench.com/) — code-fixing benchmark
- [AgentBench](https://github.com/THUDM/AgentBench) — agent capability eval
- [WebArena](https://webarena.dev/) — web agents (touches your browser-agent skills)
- [OSWorld](https://os-world.github.io/) — computer-use eval

**Projects to attempt:**
- Contribute to a public benchmark
- Build a domain-specific benchmark for your niche
- Write an eval framework for a specific tool category

## Path D: Start a micro-product

The capstone proved you can ship. Maybe it's actually a business.

**Signs your capstone could be a product:**
- People are starring the repo
- You're getting GitHub issues with real use cases
- Someone has asked "can I pay you to set this up?"

**Paths:**
- Open-source with paid cloud tier (Supabase model)
- Consulting gigs installing it at companies
- YC / accelerator application

Related reading:
- [How to start a startup (Paul Graham)](http://paulgraham.com/start.html)
- [Pieter Levels' indie hacker blog](https://levels.io/)
- [indiehackers.com](https://www.indiehackers.com/)

---

## Path E: Continue as a generalist AI Engineer

Most practical path for career change with stable income.

**Interview prep:**
- LeetCode-style isn't required but not harmful (top 50 problems)
- Practice system design for LLM apps (rag system, agent system, eval system)
- Have 2–3 project war stories ready to tell in 3 minutes each

**Where to apply (2026 hiring):**
- YC companies (high AI adoption, hiring aggressively)
- Mid-market SaaS doing "AI features" (most companies)
- Dev tool startups — high technical bar, AI-native by default

**Prep materials:**
- [AI Engineering by Chip Huyen](https://www.oreilly.com/library/view/ai-engineering/9781098166298/) (book)
- [The Full Stack LLM Bootcamp](https://fullstackdeeplearning.com/llm-bootcamp/)

## Regardless of path — keep these habits

- **Weekly build.** Even if tiny. Shipping is muscle memory that atrophies.
- **Monthly post.** Blog, LinkedIn, Twitter — any form of public writing. Your future self will thank you.
- **Quarterly audit.** Am I learning? Am I energized? If no, pivot without guilt.
- **Annual review.** Open the journal from Module 1. Marvel at past-you's confusion about things you now find obvious. Write an update letter.

**You are no longer a QA Engineer learning AI. You are an AI Automation Engineer with deep QA roots.**

That identity change compounds over years. Live into it.
